Background replacement: 3 approaches

Static random navy + orange blobs, heavy blur, manual dark toggle, per-section seeds

A

SVG Blob Component

Reusable BackgroundBlobs.astro renders inline SVG circles/ellipses. A tiny JS snippet randomizes position, scale, and opacity on every page load. CSS toggles navy/orange palette and dark mode.

CSS Variables + JS Seed

C

Canvas Background

Each section gets a <canvas> client island that draws blurred navy/orange blobs. Most flexible shapes and true randomness, but heavier JS and more runtime cost.

Trade-offs

  • A (SVG Component): Clean migration, no new deps, SSR-friendly, easy per-section seed prop. Best balance.
  • B (CSS Vars): Minimal markup change, but verbose coordinate system and harder to maintain many blob variants.
  • C (Canvas): Maximum visual flexibility, but heavier runtime and requires client islands for every section.

Recommendation: Approach A — reusable SVG component with client-side randomization and a manual dark-mode class.
